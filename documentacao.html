Documentação da API CodeHUB

Versão: 2.0 (Firebase Integration)

Visão Geral

A API CodeHUB permite integrar projetos e arquivos de código armazenados no Firebase com seu aplicativo ou site. Ela fornece métodos para:

  • Carregar projetos de um usuário específico
  • Visualizar conteúdo de arquivos de código
  • Gerenciar autenticação via Firebase

Configuração Inicial

Para usar a API, inclua o seguinte script em sua página:

<script src="https://alexandre7888.github.io/CodeHUB/API/api2.js"></script>

Parâmetros da URL

A API espera os seguintes parâmetros na URL:

Parâmetro Tipo Obrigatório Descrição
userKey String Sim* Chave única do usuário (formato: ID-TOKEN)
userName String Sim* Nome do usuário para filtrar projetos

* Pelo menos um dos parâmetros (userKey ou userName) deve ser fornecido.

Métodos da API

1. renderProjects(container, userKey, userName)

Renderiza os projetos do usuário no elemento container especificado.

Parâmetros:

  • container - Elemento DOM onde os projetos serão renderizados
  • userKey - Chave do usuário (opcional)
  • userName - Nome do usuário (opcional)

Exemplo de Uso

<!DOCTYPE html>
<html>
<head>
  <title>Exemplo de Integração</title>
  <script src="https://alexandre7888.github.io/CodeHUB/API/api2.js"></script>
        <script src="https://alexandre7888.github.io/CodeHUB/API/userKeyAPI.js"></script>
</head>
<body>
  <div id="projects-container"></div>
  
  <script>
    // A API é automaticamente inicializada quando o DOM estiver pronto
    // Certifique-se de incluir userKey ou userName na URL
    // Exemplo: ?userKey=12345-abcde&userName=Alexandre
  </script>
</body>
</html>

Estrutura de Dados

A API retorna projetos no seguinte formato:

{
  "id": "project123",
  "name": "Meu Projeto",
  "createdAt": "2023-01-01T00:00:00Z",
  "files": {
    "file1": {
      "originalName": "script.js",
      "content": "console.log('Hello World');",
      "language": "javascript"
    }
  }
}

Tratamento de Erros

A API exibe mensagens de erro diretamente no container especificado em casos de:

  • Parâmetros ausentes na URL
  • Falha na conexão com o Firebase
  • Projetos não encontrados para o usuário

Estilos CSS

A API inclui estilos padrão para os seguintes elementos:

  • Container de projetos (.ch-projects-container)
  • Cartões de projeto (.ch-project-card)
  • Modal de visualização de código (.code-modal)
  • Indicadores de carregamento e erro

Boas Práticas

  • Forneça ambos os parâmetros (userKey e userName) para melhores resultados
  • Certifique-se de que o container tenha espaço suficiente para exibir os projetos
  • Teste a integração em diferentes tamanhos de tela
  • Não modifique os estilos CSS da API diretamente - sobrescreva-os em seu próprio CSS se necessário
