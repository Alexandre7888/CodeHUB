Documentação da API OAuth – CodeHUB

Versão: 1.0 · Autor: Alexandre

Visão Geral

A API OAuth do CodeHUB permite:

  1. Criar um token de autenticação (criar.app)
  2. Continuar a sessão de um usuário autenticado (continuar-conta)
  3. Obter informações do usuário via JavaScript (userKeyAPI.js)

1. Criar Token

URL Base:

https://alexandre7888.github.io/CodeHUB/API/criar.app

Método:

GET

Descrição:

Este endpoint gera um token único para o usuário após autenticação. Geralmente redireciona para uma URL de callback ou exibe o token na tela para uso posterior.

Fluxo:

  1. O usuário acessa o link criar.app.
  2. A API autentica e retorna um token.
  3. O token pode ser usado no endpoint /continuar-conta para carregar a sessão.

2. Continuar Conta

URL Base:

https://alexandre7888.github.io/CodeHUB/API/continuar-conta?token={TOKEN}

Parâmetros:

NomeTipoObrigatórioDescrição
tokenstringSimToken do usuário criado no criar.app.

Exemplo de Requisição:

GET https://alexandre7888.github.io/CodeHUB/API/continuar-conta?token=abc123xyz

Resposta:

{

"status": "success", "user": { "id": "12345", "name": "Alexandre Braatz", "email": "alexandre@example.com" } }

3. Script – userKeyAPI.js

URL:

https://alexandre7888.github.io/CodeHUB/API/userKeyAPI.js

Função:

  • Ler o token na URL (?token=...)
  • Fazer requisição ao endpoint /continuar-conta
  • Exibir dados do usuário na página

Exemplo de Uso:

<!DOCTYPE html>

   Conta do Usuário   
Carregando...

4. Fluxo Completo

  1. Usuário cria token → Acessa criar.app → Recebe TOKEN.
  2. Frontend redireciona para página com ?token=TOKEN.
  3. Script userKeyAPI.js lê token, chama /continuar-conta, obtém dados do usuário e exibe na tela.
